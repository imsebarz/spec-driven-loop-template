# Contributing

Thanks for contributing.

## Ground rules

- Specs first.
- Keep `IMPLEMENTATION_PLAN.md` current.
- Prefer small complete increments over large partial rewrites.
- Add or update tests for every acceptance criterion you close.
- Do not leave placeholder scripts if a real command can exist.

## Recommended loop

1. Update or add `specs/*`
2. Run a planning pass
3. Run a short build loop
4. Review the diff
5. Merge only completed increments

## Pull requests

A good PR should include:
- what spec item it closes
- what tests prove it
- any plan updates that changed priority or scope
